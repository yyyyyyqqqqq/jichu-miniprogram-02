# 第五阶段云商品发布部署与验收

## 1. 确认云环境与前置数据

1. 在微信开发者工具顶部点击“云开发”。
2. 确认当前环境为：

```text
cloud1-d9gpdpv6p2db56d8e
```

3. 确认 `users`、`products` 集合仍存在。
4. 确认 `products` 集合继续禁止客户端直接读写；商品创建和读取均由云函数完成。
5. 不需要新建集合，不需要为发布链路新增索引。第四阶段查询索引保持不变。

正常结果：第三阶段登录和第四阶段商品读取仍能使用，客户端没有直接数据库写权限。

## 2. 部署 createProduct

### 开发者工具界面

1. 在文件树中找到 `cloudfunctions/createProduct`。
2. 右键目录。
3. 选择“上传并部署：云端安装依赖”。
4. 环境选择 `cloud1-d9gpdpv6p2db56d8e`。
5. 等待云函数状态变为 `Active`。

### CLI

```powershell
& "D:\program\微信web开发者工具\cli.bat" cloud functions deploy `
  --env "cloud1-d9gpdpv6p2db56d8e" `
  --paths "D:\codex\jichu mini program02\cloudfunctions\createProduct" `
  --remote-npm-install `
  --project "D:\codex\jichu mini program02"
```

正常结果：云函数列表出现 `createProduct`，状态为 `Active`。

常见失败：

- `Creating`：云端仍在创建函数，稍后刷新状态。
- 云端安装依赖失败：确认选择“云端安装依赖”，并检查 `package.json`。
- `function not found`：部署环境或函数目录不正确。

## 3. 重新编译

1. 在微信开发者工具选择“工具 → 清除缓存 → 清除全部缓存”。
2. 点击“编译”。
3. 确认控制台没有编译错误和关键运行时异常。
4. 确认首页真实商品、自定义 TabBar 和商品详情仍能打开。

## 4. 登录守卫

1. 先在个人中心退出登录。
2. 点击中间“发布”按钮。
3. 确认进入登录页，而不是直接打开发布表单。
4. 完成登录。
5. 确认安全返回发布页，页面显示商品发布表单。

正常结果：未登录用户不能提交，登录后复用现有用户会话。

## 5. 表单校验

依次只触发一种错误，确认按钮恢复可点击且 Loading 会关闭：

1. 不填标题：提示填写商品标题。
2. 标题少于 2 个字符或超过 40 个字符：提示标题长度不正确。
3. 不填描述：提示填写商品描述。
4. 描述少于 5 个字符或超过 1000 个字符：提示描述长度不正确。
5. 价格为空、为 `0`、负数、字母、三位小数或超过 `999999.99`：提示价格不正确。
6. 不选分类：提示选择分类。
7. 不选新旧程度：提示选择新旧程度。
8. 不填交易地点：提示填写交易地点。
9. 不选图片：提示至少选择一张图片。

## 6. 图片选择、预览与删除

1. 点击图片区域，选择 2 张图片。
2. 点击其中一张，确认可以预览。
3. 删除其中一张，确认剩余数量和计数正确。
4. 继续选择图片，确认总数最多为 6 张。
5. 尝试选择超过 10MB 的单张图片，确认页面阻止加入并给出提示。

数据库中不得出现本地临时路径；只有全部图片上传成功后才会创建商品。

## 7. 正常发布

建议使用容易定位的测试内容，例如：

```text
标题：第五阶段验收台灯
描述：第五阶段真实发布验收，功能正常，仅校内面交。
价格：29.90
分类：生活
新旧程度：九成新
交易地点：图书馆南门
```

1. 选择至少一张测试图片。
2. 点击“发布商品”。
3. 确认上传和保存期间按钮禁用，页面只显示一个 Loading。
4. 确认提示“发布成功”。
5. 确认表单和临时图片状态已清空。
6. 确认自动进入新商品详情；若详情跳转失败，应回到首页且不重复发布。

## 8. 查看云存储图片

1. 打开“云开发 → 存储”。
2. 进入 `products` 目录。
3. 进入以当前用户安全 ID 命名的子目录，再进入日期目录。
4. 确认本次图片文件存在，路径类似：

```text
products/{userId}/{YYYYMMDD}/{时间戳}-{序号}-{随机值}.{扩展名}
```

路径不包含明文 OPENID，文件名不只使用原始文件名。

## 9. 检查 products 数据

1. 打开“云开发 → 数据库 → products”。
2. 按测试标题搜索本次记录。
3. 核对：

```text
_id                 p_ 开头的确定性 ID
title               与输入一致
description         与输入一致
price               Number 类型，例如 29.9
categoryId/name     来自现有分类配置
condition           与选择一致
location            与输入一致
images              cloud:// 开头的 fileID 数组
coverImage          images[0]
sellerId            当前 users 文档 ID
sellerOpenid        云函数上下文对应的真实身份
sellerName/avatar   来自当前 users 文档
status              available
viewCount           0
favoriteCount       0
createdAt/updatedAt 数据库 Date 类型
```

确认没有本地临时文件对象、临时路径、`undefined` 或客户端伪造字段。

## 10. 确认卖家身份

1. 复制商品记录的 `sellerId`。
2. 在 `users` 集合查找同 `_id` 文档。
3. 对比昵称、头像和校园字段。
4. 确认商品卖家信息来自该用户记录。
5. 客户端请求只包含 `requestId` 和商品输入字段，不包含可信卖家、状态、计数或时间字段；这些字段由 `createProduct` 构造。

## 11. 重复提交与结果未知重试

1. 填写一份新的测试商品。
2. 快速连续点击发布按钮多次。
3. 发布结束后按标题查询 `products`，确认只创建 1 条记录。
4. 如遇网络超时且页面提示确认结果，可直接点击重试；不要修改表单。
5. 重试使用同一发布请求 ID，数据库中仍应只有 1 条商品。

说明：商品文档 ID 由真实用户 ID 与发布请求 ID 共同生成；服务端在重复请求时返回已有商品。

## 12. 首页与持久化

1. 从详情返回首页，或点击首页 Tab。
2. 选择“最新”排序并下拉刷新。
3. 确认新商品可见，图片、价格、地点和卖家信息展示正常。
4. 完全关闭并重新打开小程序。
5. 再次搜索该标题并打开详情，确认商品仍然存在。

## 13. 失败与 Loading

可在断网或开发者工具网络异常条件下测试：

1. 上传失败时应提示图片上传失败或网络异常。
2. 保存失败时应提示商品保存失败。
3. 所有错误分支都应关闭 Loading。
4. 错误后按钮应恢复可点击。
5. 保存明确失败时，本次已上传图片应尝试删除；在云存储中检查没有对应孤儿文件。
6. 清理失败只记录诊断警告，不应覆盖原始发布错误。

不要为测试修改并提交云函数名、环境 ID 或数据库权限。

## 14. 第三、四阶段回归

1. 首次登录仍只创建一个用户。
2. 再次登录不会创建重复用户。
3. 重启后会话恢复正常，退出登录正常。
4. 首页真实商品读取、搜索、分类、四种排序、分页、下拉刷新和错误重试正常。
5. 商品详情、`available`、`reserved`、`sold` 状态与缺失商品处理正常。
6. 自定义 TabBar 和登录后安全返回正常。
7. 数据库异常时 Loading 可以结束。

人工验收通过前不要创建 Git 提交、推送或阶段标签。
